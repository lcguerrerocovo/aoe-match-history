1. Added React + Vite + Chakra UI for the main interface
2. Kept static APM charts generation
3. Implemented dynamic match list rendering
4. Set up middleware for serving match data

## Key Components

### Vite Configuration (ui/vite.config.ts)
- Serves the React application
- Middleware for serving match data from `/data` directory
- Handles both directory listing and file serving
- Maintains static site serving for APM charts

### Match List Component (ui/src/components/MatchList.tsx)
- Renders match history in an accordion format
- Groups matches by date
- Shows match details including:
  - Match ID and link to charts
  - Start time
  - Game type
  - Map
  - Duration
  - Teams with players and civilizations

### Match Service (ui/src/services/matchService.ts)
- Fetches match data from the server
- Handles civilization mapping
- Provides functions for:
  - Getting match list
  - Getting individual match details
  - Getting civilization names

## Data Flow
1. Vite server serves the React application
2. React app fetches match data through the middleware
3. Match data is processed and displayed in the UI
4. APM charts remain static and are served directly

## Static vs Dynamic
- **Dynamic**: Main match list interface (React)
- **Static**: APM charts (generated by Python script)

## Dependencies
- React for UI
- Vite for development and building
- Chakra UI for components
- TypeScript for type safety
